# 📘 SQLPhone Emperor · SQL Module 04
# 📖 L‑37 – Join Challenges

## 🎯 OBJECTIVE
Apply all join types to solve realistic business queries.

## 🧱 BRICK 1 – Challenge Scenarios
Given these tables:
- `students` (id, name)
- `courses` (id, title)
- `enrollments` (student_id, course_id, grade)

Write queries to:
1. List all students and the courses they’re enrolled in
   (include students with no enrollments).
2. Find courses that have no students.
3. Show all possible student‑course combinations
   (students × courses) and mark actual enrollments.
4. Display a full outer join of students and courses
   based on enrollment (use simulation).
5. Use a self‑join to find students who share the same
   course (advanced).

## 🧱 BRICK 2 – Deliverable
Create the tables, insert sample data, and write all five
queries in a single `.sql` file.

## 💡 Why It Matters
These are real interview questions and daily tasks
for any backend engineer dealing with relational data.

## 📌 Key Takeaway
Joins are not just syntax – they are tools to answer
questions about relationships.
Practice with varied scenarios builds fluency.

*The challenge is not the join; it’s the question.*
