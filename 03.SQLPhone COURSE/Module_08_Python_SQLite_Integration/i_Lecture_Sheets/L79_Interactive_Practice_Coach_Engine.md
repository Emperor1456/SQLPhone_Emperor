## 🧱 ENGINE ARCHITECTURE

The practice engine follows a simple but powerful pattern:

1. Connect to a temporary in‑memory database.
2. Execute setup SQL (create tables, insert seed data).
3. Present a task description to the user.
4. Accept the user’s SQL query as multi‑line input.
5. Execute the query and capture the output.
6. Compare the output to an expected result.
7. Provide hints if the user is stuck.
8. Offer retry or advance to the next level.

## 🧱 CORE ENGINE SKELETON

```python
import sqlite3

def run_practice(description, setup_sql, expected_output, hints):
    conn = sqlite3.connect(":memory:")
    conn.executescript(setup_sql)

    print(description)
    hint_idx = 0

    while True:
        print("\nEnter your SQL (empty line to submit):")
        lines = []
        while True:
            line = input()
            if line == "":
                break
            lines.append(line)
        user_sql = "\n".join(lines)

        if user_sql.strip().lower() in (":hint", "hint"):
            if hint_idx < len(hints):
                print(f"💡 Hint {hint_idx+1}: {hints[hint_idx]}")
                hint_idx += 1
            else:
                print("No more hints.")
            continue

        if user_sql.strip().lower() in (":quit", "quit", "exit"):
            print("Practice ended.")
            break

        try:
            cur = conn.execute(user_sql)
            rows = cur.fetchall()
            output = str(rows)
            if output == expected_output:
                print("✅ Correct!")
                break
            else:
                print(f"❌ Expected: {expected_output}")
                print(f"   Got:      {output}")
        except sqlite3.Error as e:
            print(f"❌ Error: {e}")

    conn.close()
```

## 🧱 ADDING A TASK

Each practice sheet defines a list of `Task` objects (using the engine’s Task class) and passes them to `run_sequence()`. A task includes:

- `description` – what the user must do
- `setup_sql` – creates the tables and data
- `expected_output` – the exact result string expected
- `hints` – progressive clues
- `level` – Easy, Medium, or Hard
